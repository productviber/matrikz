KEY PATTERNS:

1. Design System (Shared by Both Workers)
   - All UI components import from @visibility/design-system/components
   - All styles use tokens from @visibility/design-system/tokens
   - Both workers use identical Button, Card, MetricCard components

2. Monorepo Structure (pnpm workspaces)
   - packages/design-system is dependency for both workers
   - packages/analytics and packages/marketer are independent
   - Easy to add more workers (visibility-ai, visibility-email, etc)

3. Database Integration
   - Both workers access shared D1 database
   - Shared tables: users, sites
   - Analytics-only: gsc_data, bing_data, cloudflare_data
   - Marketer-only: conversions, affiliates, email_campaigns

4. Service-to-Service Communication
   - Marketer worker calls: GET analytics/internal/report-data/:domain
   - Allows analytics to apply business logic before exposing data
   - Caching strategy: marketer caches results in KV

5. Deployment Strategy
   - Two independent workers deployed to Cloudflare
   - Routes configured at Cloudflare level (not in code)
   - Automatic scaling per worker
   - Clear traffic boundaries
